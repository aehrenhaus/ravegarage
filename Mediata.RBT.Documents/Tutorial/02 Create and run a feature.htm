02 Create and run a feature

  1. Right click the Features folder in Medidata.RBT.Common.Steps project. Add new item
  2. You shall see 3 new templates if you installed specflow
    Choose the feature file template, name it myfeature.feature

    Enter the following text

  3. Compile (F6) ,go to test view window,

    Here
    Group name = Class name = 'MyFeatureTitleFeature'
    Test =Method name = 'MyScenarioTitle' 

    Class name comes from the first line after 'Feature:'  , camelized
    Method name comes from the first line after 'Scenario:' ,camelized
  4. Run it , you will see it takes you the the url.
    'I navigate to url "" ' is a defined step.


    As a programmer you should write and maintain these steps defintions.
    A step definition is a method marked by an attribute, see examples later.

    As a SQA, you can write steps in the feature file that does or does not have a matched step definition. If it does not , find a programmer to implement the match for the step.

    A step in the scenario that does not have a step definition will not cause compiler to fail but will cause the test to fail.
