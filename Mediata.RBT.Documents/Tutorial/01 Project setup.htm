01 Project setup

  1. Uninstall Typemock 6.0.2 which is usually installed on medidata's machines, it conflicts with the version of selenium we are using.
  2. Install specflow, this will provide nessisary plugins for VS2010
  3. Clone RBT project using git
  4. Get your self a copy of firefox12. At this time, only firefox 12 works properly with the selenium dlls.
  5. In ravegarage\Medidata.RBT.Features.Rave folder, make a copy of App.config.template ,rename the new file to App.config
  6. Now you can open the solution, and edit app.config.
    1. Config your path to the copy of firefox browser
    2. Optionally you can config the path to the firefox's profile folder. There is a zipped copy in ExternalDlls folder.
      This profile contains firebug.
      With a blank profile setting, eachtime there will be a new profile generated and you won't get any of you plugins installed to other firefox.
    3. Config the rave url and database connection strings
  7. Open the solution in VS, compile it(F6), open 'test view'  if you don't see it(Menu: Test->Windows->TestView)
    There should already be many tests there .
    Click the group by dropdown to group by Class
  8. Pick any test , right click and run it. It might fail but at least you should see browser pops up trying to run the test.
    And later the report page pops up.
