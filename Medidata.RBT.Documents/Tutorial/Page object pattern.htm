Page object(PO) pattern

Page object pattern means for each page there is a class represents it.

Simple operation such as clicking a button or business level opertion such as uploading a draft can be encapsulated into a method in PO class.

With PO class in hand , we usually don't operate the page in step definitions, instead, we call methods on page object instance( the CurrentPage variable ) and let it do the operations.

 

In RBT project, all PO classes inherit from IPage, there could be some base classes in between your PO class and IPage, for example: PageBase, RavePageBase, EDCPageBase. Read these classes is helpful to understand the project.
