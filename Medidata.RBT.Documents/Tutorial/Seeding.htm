Seeding

For some tests there should be some data pre exists in the system. For example a  study and an assignment must exist in order to test a DT. The insertion of these data is called seeding.

Problem is how to insert them. There were many debates about the way to do it. I don't want to tell the whole story, but here are some options given:

  • Use sql to seed database directly. (have to know every details of how rave operates the db)
  • Use Rave's core objects. ( have to reference Rave code base)
  • Write steps in RBT to seed from UI
  • Seed manually from UI, and save the database , and restore it before every test (Too many databases to backup, also you can not use a shared site any more ,becuase you can't recover the db that others are using)

There are other problems like shall the seeded data be cleaned?  How to clean up the seeded data?

Finally one day some smart developers come up with a brilliant idea, a feasible method, and that becomes our seeding methodology today.

The idea is to seed from Rave UI with object oriented code in RBT project.  Most of seeding take advantage of Rave's uploading function. In Rave you can create Projects, Users, Sites etc by uploading xml files.

There are some presets of xml files in RBT files. When the seeding step executes, it's going to make a copy of such a file, modify it to make it unique (for example unique username for User xml) and navigate to upload page to upload the file.
